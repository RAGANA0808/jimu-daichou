## 1. 目標サービス分析 — せいざん「クラウド管理 寺務台帳 / 檀信徒カルテ™」

我々のベンチマーク（北極星）となる直接の目標サービス。「家単位ではなく個人単位の檀信徒カルテ」「2ボタンで完了する法要案内」を強く打ち出している。

### 1.1 中核となる強み（Must-have として取り込むべき）

- **個人単位の檀信徒カルテ**: 施主・親族・過去帳・見込み客（未契約問合せ者）を1カードに統合。我々は「世帯(Household)単位」が基本だが、個人単位のビューと見込み客（問合せファネル）管理が弱い。
- **関係性・交流記録（カルテ内タイムライン）**: 交流履歴・伝言・先代との約束・悩み/不安・戒名由来・先祖の歴史を時系列で記録し、住職交代・世代交代で関係資本を継承。**単なる属性DBではなく「約束・伝言・履歴」の記録欄が必須**。→ 我々の `InteractionNote` は schema 定義のみで UI 未実装（最大の欠落）。
- **ひらがな/漢字/特徴によるあいまい検索**: 顔と名前が一致しなくても、ひらがな・特徴メモから即座に呼び出す。検索ボックスを最上位導線に置く。→ 我々は `lib/search` ディレクトリが空。
- **属性別グループ設定**: 檀家/永代供養/納骨堂/見込み等でグループ化し案内対象を絞り込む。
- **法要・回忌管理**: 過去と未来の回忌を一覧表示し、年忌が来る対象を抽出。
- **法要案内の2ボタン出力（差込印刷）**: 対象を選び、宛名ラベル・案内本文・封筒を最小操作で差込印刷。**法要申込率を大幅向上させた中核機能**。
- **塔婆申込・塔婆読上帳の印刷**: 行事・法要に紐づく塔婆申込を記録し、読上順に並べた「塔婆読上帳」を出力。→ 我々に未実装。
- **月参り/棚経などの定期法務のスケジューリング**: 月参り対象の自動割り振り、スキップ機能、期日経過後の自動履歴化。棚経はドラッグ&ドロップで訪問順を調整（300件規模）。→ 我々の区画地図 DnD 資産を転用可能。
- **永代供養墓/納骨堂の区画・契約・合祀管理**: 区画の空き状況・契約情報・合祀年数（合祀予定表）・遺骨状態を一元化。
- **会計・入金管理**: 永代供養/管理費の入金状況をカルテ・区画と紐づけて把握。
- **画像・書類の添付保存** / **マルチユーザー同時編集** / **クラウド保管+日次自動バックアップ+暗号化** / **スマホ/タブレット/PC マルチデバイス対応**。
- **過去帳のデータ化（移行）支援**: 新旧住所・同姓同名・重複チェック、異体字/旧字体対応、書き起こし+人手最終確認。導入時の移行サービスとして事業上必須。

### 1.2 あると良い（Nice-to-have）

音声入力でのカルテ記録 / スマホ撮影→書類保存 / メール通知 / 他社ソフト(AXESS・FileMaker・Salesforce)からの移管 / 表示カスタマイズ / LINE サポート導線 / 寺院健全度診断 / 経営コンサル伴走 / ペット供養・樹木葬の契約管理 / 郵便番号DB定期更新 / 見込み客ファネル管理 / 導入事例・口コミ蓄積。

### 1.3 UI/UX 設計の示唆

- **「2ボタンで完了」の徹底**: 法要案内のラベル・本文・封筒印刷を2クリックで終える。操作手数そのものを売り文句にできるレベルまで削る。
- **70代住職が迷わず使える**ことを明示的な設計目標に。大きめ UI・少ない選択肢・専門用語回避。
- **顔と名前が一致しない現場課題に直結する「ひらがな/特徴あいまい検索」**を最上位導線に。特徴メモを検索対象に含める。
- **「期日経過で自動的に履歴化」**のように、ユーザーが消し込まなくても状態が前に進む仕組みで入力漏れ・対応漏れをゼロにする。
- **面会前プレビュー UX**: カルテを開いた瞬間に「前回の困りごと・約束・家族背景」が一覧で見える。
- **効果を数値で語る訴求**: 行事案内が5日→2時間、法要申込率80%超、伝え漏れゼロ。
- **永代供養/納骨堂は「区画の空き×契約×合祀年数×入金状況×遺骨状態」を1画面で横断把握**。地図・会計・カルテを分断せず統合ビューにする。

## 2. 競合製品（寺院管理・檀家管理ソフト）

天空21 / お寺まもる君 / お寺プラス / 一如 / 蘇婆訶 / ガンジス「ご縁帳」 / 寺Pay / しゅうごう(LINE) / 彩雲(kintone) / ベストプラクティス(Salesforce) ほか。

### 2.1 競合が標準装備する Must-have 機能

- **戒名重複チェック・戒名作成支援**（天空21標準）: 戒名・法名入力時に過去帳内の重複や類似を警告し、院号・道号・戒名・位号の構成を補助。→ 我々の過去帳は戒名を単一テキストで持ち、構造化・重複チェック未搭載。
- **年忌対象者の自動抽出+宛名印刷ワークフロー**（天空21/お寺まもる君/蘇婆訶 共通）: 抽出→案内状文面生成→宛名ラベル/はがき/封筒印刷まで一気通貫。→ 我々は年忌表PDFはあるが「発送業務」が抜けている。
- **中陰（四十九日まで忌日）計算と中陰表出力**: 初七日〜七七日・百ヶ日の自動算出と中陰表印刷。年忌計算とは別の必須標準機能。→ 我々に未実装。
- **施主・親族・故人の関係性を紐づけた一体管理**（お寺まもる君「一体型」）: 1つの墓・家に複数故人と施主・継承者を関連付け、家系として辿れる。
- **管理費・護持会費の年次請求と入金消込・未納管理**: 区画ごとの年会費を一括請求生成し入金状況を可視化。→ 我々は会計CRUDのみで債権管理サイクル未整備。
- **宗教法人提出書類・財務帳票の出力**（天空21）: 収支計算書・財産目録など宗教法人法対応の帳票。
- **Excelインポート/エクスポート**（一如・お寺プラス）: 既存名簿・過去帳の一括取込と書き出し。乗り換え障壁を下げる導入必須要件。
- **あいまい検索・横断検索**（天空21）: 氏名/住所/戒名の一部・うろ覚えでもヒット。檀信徒・過去帳・法要を横断。
- **スマホ/タブレット最適化された外出先閲覧**（お寺プラス）。

### 2.2 Nice-to-have（差別化候補）

お布施・管理費のキャッシュレス/オンライン収納代行（寺Pay型、檀家の56%がキャッシュレス希望） / 檀家向け LINE・ポータル配信（しゅうごう型） / 法要・塔婆のオンライン申込フォーム / 関係性・対話ログ記録（ガンジス「ご縁帳」型） / 墓地図ワンクリック逆引き・印刷用案内図（蘇婆訶型） / 命日・回忌の自動リマインド / kintone・Salesforce 型のカスタムフィールド拡張 / デジタル御朱印・参拝者エンゲージメント / 手書き名簿のデジタル化代行。

### 2.3 UI/UX の示唆

- **うろ覚え前提の検索を最優先**: 「〇〇さんの隣の墓の家」「戒名の一部だけ」状態で引く。部分一致・かな/漢字ゆらぎ吸収を検索ボックス1つで。
- **宗派差の吸収**: 年忌の数え方・呼称（門徒/檀家、法名/戒名）・中陰の作法が宗派で異なる。宗派プリセット切替で初期設定負担を下げる。
- **発送業務を1画面の流れに**: 年忌抽出→宛名→印刷を「今月の年忌案内を出す」という1タスクとして導線化。
- **データ移行のしやすさを入口に**: Excel取込と入力代行で乗り換え障壁を下げる。
- **キャッシュレスは「非課税・信教の自由」への配慮を明示**: 檀家手数料無料・データ非共有を打ち出す。
- **「記録して終わり」でなく「活用・関係維持」を見せる**: ダッシュボードで「今月の命日・疎遠檀家・未連絡先」などアクションにつながる気づきを提示。

## 3. 宗教法人会計・お布施・護持会費の業界要件

宗教法人法・文化庁様式に準拠した会計が求められる。現行の単純な入出金 CRUD には**勘定科目体系・口座区分・年度締め・債権管理・帳票出力が決定的に不足**している。

### 3.1 Must-have

- **勘定科目マスタ**: 収入（布施・御供・護持会費・維持会費・永代供養料・志納金・祈祷料・寄付金・資産運用収入・補助金収入）/ 支出（法要費・儀式行事費・教化布教費・修繕費・人件費・公租公課・会議費・事務費 等）。文化庁の収支計算書様式例の科目セットと自法人カスタム様式の2系統を切替可能に。
- **現金と預金を物理的に分離した複数口座・複数現金箱管理**（本堂賽銭・庫裏現金等）。取引ごとに対象口座を指定し、口座別の出納帳と残高を持つ。
- **現金出納帳・預金出納帳の自動生成**（日付・科目・摘要・相手先・入金/出金/残高、口座別/期間別フィルタ、Excel/PDF出力）。
- **収支計算書の自動作成**（文化庁様式・標準様式）。**会計年度終了後3か月以内作成**という法定スケジュールに合わせた年次締め。
- **財産目録の作成・管理**（基本財産・普通財産・什物・墓地・建物等、年1回棚卸、3か月以内作成、所轄庁へ4か月以内提出）。
- **護持会費（会費）台帳**: 檀信徒カルテ連携で世帯ごとに年額会費・賦課額・納入区分（年払/盆彼岸集金/口座振替/手集金）を登録し、年度ごとの請求対象を一括生成。
- **護持会費の入金消込**: 世帯単位で納入済/未納/一部納入を判定。入金登録と同時に会計収入へ自動計上し二重入力を排除。
- **未収金（未納）管理と督促**: 年度・世帯別の未納一覧、未納額・滞納年数の集計、督促状の差込印刷・送付履歴記録。
- **領収書の発行**: お布施・護持会費・寄付に対し宛名・但し書き・金額・発行日入りの領収書を発行し履歴保持。
- **収益事業と非収益事業の区分経理**: 取引に事業区分タグを付与し区分別集計。法人と住職個人の収支を明確に分ける。
- **年次決算の締め処理**: 会計期間ロック、前年度繰越、年度比較、期中の遡及修正統制。
- **監査・証憑トレーサビリティ**: 各取引に証憑（領収書・通帳画像）添付、誰がいつ登録・修正・削除したかの変更履歴（監査ログ）。

### 3.2 Nice-to-have

予算管理とダッシュボード（予算対実績グラフ） / レシート・通帳のOCR取込 / 貸借対照表・正味財産増減計算書 / 口座振替・コンビニ/クレジット/QR納入と自動消込 / 源泉徴収・人件費補助 / 区画と護持会費請求の連動 / 法要記録からの入金自動起票 / 電子帳簿保存法対応 / 本山への宗費報告様式 / 総会向け会計報告（円グラフ・前年比） / 消費税区分。

### 3.3 UI/UX の示唆

- 入力者は会計知識の乏しい住職・寺族・事務員。**借方貸方を見せず「出納帳メタファ」**（日付・いくら・何の費目・誰から/誰へ）に徹し、複式簿記は裏側で自動処理。
- 護持会費は**世帯ごとの年次集金管理**が業務の中心。カルテ1画面から当年の請求額・入金状況・未納を確認でき、入金が即会計収入に反映される一気通貫導線。
- **未納の見える化**: 年度切替で「今年まだ集めていない世帯」が一覧・件数・金額で即把握でき、ワンクリックで督促状の宛名差込印刷へ。
- 科目選択は寺院特有の語彙を最初から候補表示。サジェスト+過去入力の学習で誤分類防止。
- 領収書は入金記録から1クリック発行（但し書きテンプレ「護持会費として」「御布施として」）。
- 年度末の決算・所轄庁提出のリマインドと1ボタン様式出力のウィザード化。
- 現金箱と通帳を分けて扱い、残高照合支援を用意。改ざんできない記録（編集履歴）と見せても良い収支報告出力を分ける。

## 4. 墓地台帳・区画・墓じまい・永代供養の業界要件

墓埋法（墓地、埋葬等に関する法律）施行規則が管理者に備付けを義務づける帳簿に対応する必要がある。我々の DnD 区画地図は資産だが、**台帳レコード・埋蔵者・改葬・合祀・無縁化のライフサイクル管理が未実装**。

### 4.1 Must-have

- **墓地台帳（区画原簿）**: 各区画レコードに「所在区画番号・面積(平米)・墓石/外柵の状況」と「墓地使用者(名義人)の住所・氏名」を必須項目で保持。既存の区画地図(DnD)の各区画にこの台帳レコードを1:1で紐付ける。
- **使用許可・契約管理**: 使用許可証番号・許可(契約)年月日・永代使用料・年間管理料額・契約区分（一般墓/永代供養/樹木葬/合葬等）を記録し、使用許可証(個人票)を PDF 発行。再発行履歴も残す。
- **埋蔵者(納骨者)台帳**: 1区画に複数の埋蔵者をぶら下げ、死亡者氏名・続柄・死亡年月日・埋蔵年月日・焼骨/改葬の別を記録。**檀信徒カルテ・過去帳の故人レコードと相互リンクし二重入力を排除**。→ schema に `Burial` モデルが存在しないのが最大の欠落。
- **改葬(墓じまい)ワークフロー**: 「埋蔵(埋葬)証明書」を当寺が管理者として台帳データから自動発行。遺骨1体=1申請、改葬先情報、改葬許可証番号・改葬年月日を記録し、区画ステータスを使用中→返還/墓じまい完了に遷移。
- **永代供養・合祀管理**: 契約区分・安置開始日・契約安置期間（17/33/55回忌や5年・10年）と「合祀移行予定日」を保持。期限到来をダッシュボード/カレンダーで通知し、合祀後は**「取り出し不可・個別判別不可」の不可逆ステータス**で記録。
- **年間管理料の請求・入金・滞納管理**: 区画ごとに管理料年額・請求対象年度・入金/未納を管理し会計と連動。催告状 PDF 発行と送付履歴を証拠として残す。
- **名義変更(承継)管理**: 申請→承認フロー、旧/新名義人・続柄・変更年月日・添付書類（戸籍/住民票/同意書/念書）チェックリスト、承継の連鎖を時系列で保持。
- **無縁化(使用権失効)管理**: 長期滞納・名義人死亡/所在不明を「無縁化候補」として抽出。官報公告・立札掲示の周知開始日と1年間の異議申立期間をタイマー管理。異議申立中は新規埋蔵・改葬・管理料徴収を停止するロック。
- **空き区画検索**: 契約区分・面積・ステータス（空き/予約/使用中/返還/無縁化中）で検索し新規契約割当をその場で。
- **区画ステータスの一元管理**: 空き→予約→使用中→滞納→無縁化候補→返還/墓じまい完了→再貸出可、のライフサイクルを地図・台帳・会計・法要すべてに反映。

### 4.2 Nice-to-have

改葬許可申請書の下書きPDF生成 / 区画ごとの墓石・現況写真添付 / 管理料の口座振替・キャッシュレス自動収納 / 各種期限の自動リマインド / 面積からの料金自動算定 / 区画図面のA3/A4多面付け印刷・墓参者向け案内図 / 墓じまい切替シミュレーション / 親族向け閲覧ポータル / GIS連携・QR/プレート番号からの呼び出し / 無縁改葬の法定プロセス進捗の監査ログ。

### 4.3 UI/UX の示唆

- **区画地図(既存DnD)をハブにする**: 区画クリック→台帳/埋蔵者/管理料/契約期限がワンビュー。色分け（空き=灰/使用中=緑/滞納=橙/無縁化中=赤/合祀済=青）で現況が一目で。
- **二重入力を作らない**: 埋蔵者は檀信徒カルテ・過去帳の故人から選択して紐付ける。
- **不可逆操作（合祀実行・無縁改葬・区画返還）**は確認ダイアログ+理由・実施者・日付の記録を必須に。合祀は「取り出し不可」を明示警告。
- 墓じまい/改葬/承継は**手続きの進行段階が見えるステップ式（ウィザード）**に。役所提出は寺ではなく使用者である点を画面上で明示。
- 管理料の滞納は「何年分・いくら/最終連絡日/次アクション」を一覧でソート可能にし、督促対象を一括抽出→催告状一括PDFまで2クリックで。
- 専門用語は実務語（改葬許可証/埋蔵証明/合祀/承継）で統一し、入力は選択式・既定値多用。
- 契約期限は法要の Google カレンダー連携と同じ通知導線に載せる。監査・証跡を各レコードに残す。

## 5. 檀信徒コミュニケーション・お知らせ配信

「記録して終わり」でなく関係維持・案内発送のループを閉じる。**高齢の檀信徒が主対象のためデジタル一本化は不可。郵送とデジタルの自動仕分けが鍵**。

### 5.1 Must-have

- **連絡先・配信チャネル台帳**: カルテに「優先連絡手段」（郵送/電話/メール/SMS/LINE）と各連絡先・配信可否フラグを保持。1世帯=複数連絡先（当主・後継者・別居家族）、休眠/転居/到達不能ステータス。
- **セグメント抽出による宛先リスト作成**: 地区・区画・役職・年回該当・休眠/現役・任意タグで絞り込み、保存・再利用。
- **一斉お知らせ作成・配信（マルチチャネル）**: 1本文を優先チャネルへ一斉送信。デジタル不可者は「郵送対象リスト」へ自動振り分け。
- **配信テンプレート**: 行事案内/法要案内/寺報/訃報/年忌案内の用途別定型文+カルテ差し込み項目（戒名・故人名・回忌・命日・施主名）。
- **郵送向け宛名・案内状出力**: 宛名ラベル/タックシール用CSV・PDF、往復はがき・返信ハガキの差込印刷。年忌表PDFと同系統の帳票。
- **行事・法要の出欠(参加)受付フォーム**: 出席・欠席・人数・塔婆希望本数・表記・お布施項目を収集し、カルテ/法要レコードに紐づけ集計。
- **出欠・申込の集計管理画面**: 出席者数・塔婆本数・未回答者一覧をリアルタイム集計し、未回答者だけに再通知。
- **命日・年忌・行事の自動リマインド配信**: 過去帳・年忌表データで命日前/法要日前に自動リマインド（リード日数設定可）。
- **配信履歴・到達ステータスの記録**: 誰にいつ何を送ったか（チャネル・成否・開封/既読・郵送済）をカルテのタイムラインに残す。
- **訃報・緊急連絡の配慮された配信**: 一律送信でなく対象を絞り、文面を区別。誤送信防止の送信前プレビュー・確認ステップ必須。
- **マルチテナント分離** / **配信停止・同意管理（オプトアウト）**: 特定電子メール法・個人情報保護の観点で必須。

### 5.2 Nice-to-have

LINE公式アカウント連携（登録用QR・セグメント配信・リッチメニュー） / オンライン予約受付（法事・塔婆・永代供養相談、ダブルブッキング防止） / 開封率・反応ダッシュボード / オンライン志納金・お布施決済 / 下書き予約配信 / LINE版プチ寺報 / 双方向チャット / アンケート / AI下書き支援 / 高齢者向け大きな文字・UDフォントの案内状テンプレート。

### 5.3 UI/UX の示唆

- 主役は高齢の檀信徒。**「デジタル送信できる人」と「郵送する人」を1回の操作で自動仕分け**し、郵送対象は宛名出力まで一気通貫。
- 訃報・法要案内は誤送信が致命的。送信前に必ず宛先件数・チャネル内訳・差し込みプレビューを見せ確認チェックを挟む。
- テンプレ選択→対象セグメント選択→プレビュー→送信の**最小3〜4ステップのウィザード型**。
- 既存資産（カルテ・過去帳・年忌表・法要カレンダー）を起点に「このリストに案内を送る」を1クリックで。
- 出欠/塔婆申込フォームは檀信徒側もスマホで完結、氏名・故人名は既存カルテから候補補完。
- 送った記録がカルテのタイムラインに残り、電話対応時に「前回○○をご案内済み」が即分かる。
- 繁忙期（盆・彼岸・年末）向けに予約配信と「前回と同じ宛先・文面で再送」のリピート機能。

## 6. 高齢者向け業務システムの UI/UX 指針

住職・寺族は高齢者が多い。これは全機能に横断的に適用される最重要の非機能要件。CLAUDE.md §1・§4.4 と整合。

### 6.1 Must-have

- **文字拡大**: 基準16px以上・line-height 1.5以上。アプリ内で本文サイズを「標準/大/特大」3段階切替（ブラウザズーム任せにしない）。
- **コントラスト**: WCAG AA（本文4.5:1・大きい文字3:1）以上で全画面統一。**色だけで状態を表すUIを撤廃**し、文字ラベル+アイコンを併用（会計の入金/出金を色のみで区別しない）。
- **タッチターゲット最小44×44px**、ボタン間に十分な余白。区画地図DnD・カレンダー・会計CRUDの全操作に適用。
- **全アイコンにテキストラベル併記**（ハンバーガー・歯車・共有・編集鉛筆・ゴミ箱）。
- **ナビゲーション階層を浅く・一貫化**: 主要機能へトップから常時2タップ以内。パンくず+大きな「戻る」「ホーム」を常設。
- **1画面1タスク設計**: 項目数の多いフォームは縦スクロールのみのステップ式ウィザード（次へ/戻る）+進捗表示（3/5）。
- **破壊的操作に確認ダイアログ+取り消し(Undo)**: 削除は即時物理削除でなくゴミ箱/論理削除+復元導線。
- **入力中の自動保存(autosave)と下書き保持**。
- **インラインバリデーション+エラーサマリ**: エラー文は原因と直し方を具体的に（「日付は2026/06/05の形式で」）。色だけでなくアイコンと文言。
- **入力負荷軽減**: 現実的なデフォルト値・自動入力（テナント情報/今日の日付/前回値）、和暦⇄西暦の自動変換、定型値は選択式。
- **音声入力対応**: 過去帳備考・カルテメモ・法要連絡事項にマイクボタン。
- **金銭確定操作は保存前の「確認画面」を挟む2段階**（WCAG 3.3.4）。
- **スクリーンリーダー対応**: ラベル関連付け、aria-describedby、代替テキスト、キーボードのみで全操作完結。
- **現場運用向けタブレット最適化**: 屋外でも見やすい高コントラスト、横/縦持ち両対応、DnDをタッチで確実に。

### 6.2 Nice-to-have

ダークモード/ハイコントラストモード / DnDの代替操作（タップ2回での割当） / オンボーディング・文脈ヘルプ（「使い方を見る」） / 操作フィードバック強化（保存完了トースト・処理中インジケータ） / 広範なUndo・アクティビティ履歴 / オフライン入力+復帰時同期 / 印刷物も大きめフォント・高コントラスト / カレンダーの分かりやすいリマインド / 検索の入力補完（サジェスト・ふりがな・表記ゆれ吸収） / 大きな数字キーパッド・3桁区切り自動表示。

### 6.3 設計思想

- **「認識＞想起（recognition over recall）」を徹底**: 手順や記号を覚えられない前提で、毎画面でラベル・選択肢・サジェストを提示。
- 高齢者は試行錯誤・深い階層を嫌う。重要機能はホームに大きく並べ、3ステップ以上潜らせない。
- **音声入力は高齢層と相性が極めて良い**（介護現場で記録時間2割短縮）。長文フィールドに用意すると定着率・記録速度が向上。
- 「大きなボタン+音声入力+シンプルUI」の組合せが定着率を左右（介護現場で定着率85%まで向上の報告）。
- **色のみで情報を伝えない**: 状態は文字+アイコン+色の三重で冗長化。
- 余白を恐れず1画面の情報量を抑える。長い処理はステップ分割し各段で保存ポイント。
- 確認ダイアログは「はい/いいえ」でなく具体的な動詞ラベル（「この檀家を除外する」「キャンセルして戻る」）。
- テストは**実際の住職・寺族（高齢当事者）を交えたユーザビリティテスト**で。スクリーンリーダー・キーボードのみ・タブレット実機を含める。
- 和暦・旧字体の戒名・宗派固有語をフリーテキストでなく選択・変換支援で吸収。
