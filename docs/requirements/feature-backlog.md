# 寺務台帳 機能バックログ（優先度付きエピック）

本書は (A) 既存実装監査 と (B) 競合・業界・UI/UX リサーチ（`competitor-analysis.md`）を突き合わせたギャップ分析の成果物。各エピックは Phase1（自寺運用）の即効性 × 既存資産との親和性 × 実装コストで優先度付けしている。

最終更新: 2026-06-05

## 優先度・規模の定義

- **優先度**: P0=最優先（Phase1の価値の核・北極星に直結） / P1=高（業界必須・Phase1スコープ） / P2=中（差別化・Phase1後半〜Phase2） / P3=低（将来・Nice-to-have）
- **規模**: S（数日） / M（1〜2週） / L（3〜4週） / XL（1か月超・要設計）
- **現状**: 未着手 / 一部実装 / 改善

## エピック詳細

### E01 — 横断あいまい検索（名前かな・電話・戒名）
- **優先度**: P0 / **規模**: M / **カテゴリ**: 檀信徒/検索 / **現状**: 未着手
- **概要**: 「電話が鳴った瞬間に相手を引ける」北極星の中核。ヘッダー常設のグローバル検索バーから、ひらがな・漢字・電話番号の一部・戒名・俗名で世帯/故人を横断検索。表記ゆれ（ハイフン有無・かな/漢字）を吸収し、インクリメンタルサーチで即表示。`lib/search/` を新規整備。
- **受け入れ条件**:
  - [ ] ヘッダー全画面に検索バーを常設し、2タップ以内で結果到達
  - [ ] nameKana の前方/部分一致、phone/mobile のハイフン正規化一致で世帯がヒット
  - [ ] 戒名・俗名で過去帳エントリがヒット（横断）
  - [ ] schema の `@@index([tenantId, nameKana])/[tenantId, phone]/[tenantId, mobile]` を活用し withTenant 経由
  - [ ] サジェスト・入力補完（recognition over recall）
  - [ ] 結果0件時の親切な空状態
- **関連既存ファイル**: `apps/web/src/lib/search/`(空), `features/danshintoto/queries.ts`, `features/kakochou/queries.ts`, `app/(main)/layout.tsx`, `app/(main)/danshintoto/page.tsx`, `prisma/schema.prisma`

### E02 — 対応履歴タイムライン（InteractionNote）
- **優先度**: P0 / **規模**: M / **カテゴリ**: 檀信徒 / **現状**: 一部実装（schema のみ）
- **概要**: せいざんの最大訴求であり CLAUDE.md 北極星の中核「電話が鳴った瞬間に過去の履歴を把握できるカルテ」。schema 済みの `InteractionNote`（kind: PHONE/VISIT/NOTE 等・content・occurredAt・authorId）に UI・Server Action・query を実装。世帯詳細に時系列タイムラインを表示し、面会前プレビューを実現。
- **受け入れ条件**:
  - [ ] 世帯カルテに対応履歴の作成（種別選択・本文・日時・記録者自動）・一覧（時系列降順）
  - [ ] 編集・除外（論理削除）。物理削除はしない
  - [ ] 長文メモに音声入力導線（E19 と連携、後追い可）
  - [ ] タグ/ラベル付け（属性別抽出の布石）
  - [ ] withTenant + assertValidUuid 遵守、個人情報をログに出さない
  - [ ] カルテを開いた瞬間に直近履歴が見える（面会前プレビュー）
- **関連既存ファイル**: `prisma/schema.prisma`(InteractionNote定義済), `features/danshintoto/`(新規 notes), `app/(main)/danshintoto/[id]/page.tsx`
