## 決めたこと・気づき

- **「座標手入力」は UI として成立しない**。CAD ツールでもない限りマウスで配置する以外の操作は成立しないので、DnD を前提にすべきだった
- **既存の `positionX/Y` カラムは流用**。areaId NULL の行を「未配置パレット」扱いにすることで、データ移行なしで切替可能
- **Google Calendar 連携はベストエフォートで切り離す**。外部 API 失敗で DB 書込がブロックされると運用が壊れるので、DB コミット後に sync 試行 → 失敗は無視の方針
- **DnD クリック/ドラッグ判別**: `PointerSensor.activationConstraint.distance: 4` を必ず設定。これを忘れるとクリック遷移とドラッグが混線する
- **楽観的 UI は `useOptimistic` + `useTransition` の組み合わせ**。フォーム Action (`useActionState`) とは別設計で、軽量 API 呼び出しに向く
- **typedRoutes × 動的ルート** は **テンプレートリテラル必須** (`` `/kukaku/${id}` ``)。`href={{pathname, params}}` は型エラー
- **React 19 + Prisma 5 の Windows ロック問題**: dev server 起動中は `query_engine-windows.dll.node` が占有されるので、マイグレーション前に dev server を止める (`taskkill //F //PID <nextjs>`)
