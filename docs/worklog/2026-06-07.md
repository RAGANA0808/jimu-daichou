### `use server` export ビルドエラーの修正（実体は /kukaku/[id]）

`pnpm --filter @jimu-daichou/web build` が `A "use server" file can only export async functions, found object.` で失敗していた件を調査・修正。

- **実際の原因ファイル**: `features/kukaku/burial-actions.ts`（`/gojikai/daichou` ではなく `/kukaku/[id]` ページ経由で表面化）。`'use server'` ファイルなのに `export const initialBurialFormState = { status: 'idle' }`（オブジェクト定数）を export していたため、モジュール全体の export 検証で落ちていた。**環境（DB 無し）由来ではなく実コードのビルドエラー**。
- **修正（最小・規約準拠）**: gojikai / 区画の他フォームと同じく、フォーム state 型と初期値は `types.ts`（非 'use server'）へ集約。
  - `features/kukaku/types.ts`: `BurialFormState` 型と `initialBurialFormState` を追加。
  - `features/kukaku/burial-actions.ts`: 型/定数を削除し `import type { BurialFormState } from './types'` に変更（※その後、別途進行中の認証 capability + 監査ログ移行により本ファイルは `requireCapability` + `recordAudit` へさらに更新された。私の import 変更は維持されている）。
  - `features/kukaku/BurialForm.tsx`: `initialBurialFormState` の import 元を `./burial-actions` → `./types` へ。
- **検証**: 全 'use server' ファイルを grep し、非 async の値 export が他に無いことを確認（違反ゼロ）。修正後ビルドは「コンパイル成功 → 型チェック通過 → ページデータ収集通過」まで到達し、`/kukaku/[id]` も正常にビルドされるようになった（= 当該エラーは解消）。

- **build がまだ完走しない理由は別件（並行リファクタ）**: 本タスクとは別に、UI の shadcn/ui + Radix 化（`@radix-ui/react-dialog` / `@radix-ui/react-select` / `clsx` / `cva` / `tailwind-merge` 追加）と kaikei クロス集計エクスポート（`papaparse` / `xlsx` 追加）、および認証 capability + 監査ログ移行が**同時進行中**（作業ツリーに 40+ の未コミット変更、layout.tsx / app/layout.tsx / globals.css / package.json も含む）。build は静的エクスポート段階で `/nenki` 等 ~10 ページが `TypeError: Cannot read properties of null (reading 'useContext')` で失敗するが、これは**新規追加された Radix コンポーネントが prerender 中にコンテキスト不在で評価される副作用**であり、use-server 修正とは無関係・force-dynamic でも直らない。React の重複は無し（`react@19.0.0` 1 つのみ）。この prerender エラーは進行中の UI 移行ワークストリーム側で解消すべきもので、本タスクのスコープ外（該当ページ/レイアウトを他プロセスが書き換え中のため、こちらからは触らない）。
- `pnpm lint` はプロジェクトに ESLint 設定ファイル (`.eslintrc*`) が一切無く `next lint` が初期設定の対話プロンプトで落ちる**既存の未整備**。今回の変更とは無関係で、ESLint 設定の新規作成はリポジトリ全体に影響する別件のためスコープ外とした。
