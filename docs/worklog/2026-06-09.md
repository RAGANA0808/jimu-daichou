### Wave OUTREACH-NOTIFY / T-2 カレンダー取込(Google→寺・手動)（Dynamic Workflow・ultracode）

設計2並列→実装3直列→検証(レビュー3並列)。**新規モデル/ migration 不要**(既存 TempleEvent に取り込む)。ブランチ `wave/outreach-t2`。

#### 実装
- `lib/google/calendar.ts` に `listCalendarEvents(refreshToken, {timeMinISO,timeMaxISO})` + `GoogleCalendarEvent` 型を追加(events.list・singleEvents・終日/時刻の startAt/endAt 正規化・既存 export 不変)。スコープ `calendar.events` は読取も可で追加スコープ不要。
- `features/calendar-import/`(types/queries/actions): `getCalendarImportData({months})` は未連携で `{connected:false}` を返し、連携済みは [now, now+12ヶ月] の予定を取得し既存 `googleCalendarEventId`(TempleEvent/MemorialService)と突合して `alreadyLinked`/`linkedLabel` を付与。`importCalendarEventsAction` は `requireCapability('create')`・**クライアントは選択 ID のみ信用し内容は Google から再取得**・既存突合で二重作成防止・件数のみ監査(PII 非載)・TempleEvent 作成。
- UI: `/houyou/torikomi`(連携状態で出し分け: 未連携=案内+設定リンク+「自動同期しない」明記 / 連携済=未紐付けはチェックボックス・紐付け済みは選択不可+バッジ)。`/houyou` の「寺の行事」に「カレンダーから取り込む」導線。
- **特許回避(Classix JP7291923)**: 「抽出→人手選択→生成」の手動トリガのみ・自動同期パイプラインなし・外部送信/決済/オンライン参拝の連結なし・取込は Google→寺の内部処理のみ(読取専用)。
