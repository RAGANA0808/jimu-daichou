# 作業ログ (worklog)

このディレクトリには、開発セッションごとの作業ログを日付ファイルで保存します。

## 目的

- **セッションを途中で切っても次回迷子にならない**ための引き継ぎメモ
- 将来の自分や他の開発者（SaaS 化後の協力者）への「何をしたか・なぜそう決めたか」の記録
- git log だけでは追えない「判断の文脈」を残す

## 運用ルール

- ファイル名: `YYYY-MM-DD.md`（例: `2026-04-22.md`）
- 1 日 1 ファイル。同じ日に複数セッションあれば追記
- セッションの**節目**（マイグレーション完了、機能実装完了、大きな判断時）で Claude が自動的に更新
- セッション終了時にも Claude が「次回やること」を末尾に追記

## 各ファイルの構成

```markdown
# YYYY-MM-DD

## 今日やったこと
- ...

## 決めたこと・気づき
- ...

## 次回やること
- ...

## 未解決・保留事項
- ...
```

## SessionStart フックとの連動

`.claude/settings.json` の SessionStart フックにより、**最新の worklog が次回セッション開始時に自動で Claude のコンテキストに注入**されます。これで「今どこまで進んだ？」を毎回聞く必要がなくなります。
